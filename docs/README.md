Design notes, decision records, and interview-defense write-ups for the
trading_sim project.

Suggested contents as the project grows:

- `architecture.md` — component diagram + data flow.
- `decisions/` — short ADRs (one file per non-trivial design choice, with the
  trade-offs, so each can be defended cold).
- `benchmarks/` — latency results and methodology.
